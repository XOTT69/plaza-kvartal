# 🏠 Мій Дім — PWA для мешканців будинку

Progressive Web App для багатоквартирного будинку. Встановлюється на телефон як звичайний додаток.

**Проект:** `plaza-68f96` (Firebase project)

## 📱 Можливості

- **🔐 Вхід по квартирі + код** (через Firebase Auth, безпечно)
- **🏠 Головна** — останні новини та ваші проблеми
- **📢 Оголошення** — адмін публікує важливі новини
- **🆘 Аварійні контакти** — телефони з можливістю дзвінка
- **🗳 Голосування** — створення та участь у голосуваннях
- **📌 Сусідські оголошення** — "віддам", "шукаю", "знайшов"
- **⚠️ Проблеми** — повідомити про проблему, статуси: нове → в роботі → вирішено
- **📅 Події** — календар подій будинку
- **⚙️ Адмінка** — управління квартирами, контактами, оголошеннями
- **📱 Встановлюється на телефон** як звичайний додаток (PWA)

## ⚠️ Про безпеку

**Чи безпечно заливати ключі Firebase на GitHub?**

Так, **це нормально**. Firebase API ключі — публічні за дизайном, вони призначені для клієнтського коду (браузера). Справжній захист даних — у **Firestore Security Rules**.

**Як захищені дані:**
1. **Firebase Authentication** — кожен мешканець отримує свій акаунт (email = `apt-{номер}@plaza-68f96.firebaseapp.com`)
2. **Firestore Security Rules** — перевіряють, хто до яких даних має доступ:
   - Адмін може все
   - Мешканець бачить тільки свої проблеми
   - Ніхто не може читати коди інших квартир
3. **Код доступу** — це пароль від Firebase Auth

Файл з правилами: **`firestore.rules`** — скопіюй його в Firebase Console.

## 🔧 Повна інструкція з налаштування

### 1. Онови конфіг Firebase (вже готово!)

У `js/config.js` вже вставлені твої ключі. Нічого міняти не потрібно.  
Файл `config.js` містить:
- `apiKey`, `authDomain`, `projectId` — все налаштовано на `plaza-68f96`

### 2. Увімкни Authentication у Firebase Console

1. Перейди на [https://console.firebase.google.com](https://console.firebase.google.com) → проект **plaza-68f96**
2. Ліворуч обери **Authentication** → **Sign-in method**
3. Увімкни **Email/Password**
4. Залиш "Enable" (create, update, delete)

### 3. Створи базу даних Firestore

1. Ліворуч обери **Firestore Database** → **Create database**
2. Обери "Start in test mode" → **Next**
3. Обери регіон найближчий до тебе (наприклад, `europe-west3`)

> ⚠️ Потім заміни правила на безпечні (крок 4).

### 4. Додай Security Rules

У Firebase Console: Firestore Database → **Rules**

Скопіюй вміст файлу **`firestore.rules`** (він у проекті) і встав замість стандартних правил.  
Або натисни "Publish" після вставки.

**Коротко про правила:**
- Без авторизації нічого не працює
- Адмін може керувати всім
- Мешканець читає оголошення, контакти, голосування
- Мешканець створює тільки свої проблеми та сусідські оголошення
- Мешканець не бачить коди інших квартир

### 5. Створи колекції в Firestore

Після створення бази даних — натисни **Start collection** і створи:

---

#### 📁 Колекція `apartments` (Квартири)

Найважливіша колекція! Тут зберігаються коди доступу.

**Створи документи для кожної квартири:**

| Крок | Дія |
|------|-----|
| 1 | Натисни **Start collection** |
| 2 | Введи ID колекції: `apartments` |
| 3 | Введи ID документа: `1` (номер квартири) |
| 4 | Додай поля: |

| Поле | Тип | Значення | Приклад |
|------|-----|----------|---------|
| `code` | `string` | Код для входу | `1234` |
| `name` | `string` | Назва (ім'я або квартира) | `Квартира 1` |
| `isAdmin` | `boolean` | Чи адмін | `true` (для тебе) |
| `email` | `string` | Автоматично | `apt-1@plaza-68f96.firebaseapp.com` |

**Для твоєї квартири (адмін):**
```json
{
  "code": "1234",    // твій код
  "name": "Квартира 1",
  "isAdmin": true,
  "email": "apt-1@plaza-68f96.firebaseapp.com"
}
```

**Для інших 23 квартир** — ті самі поля, але `isAdmin: false`, і свої коди.

> 💡 **Важливо:** ID документа = номер квартири (1, 2, 3... 24)

---

#### 📁 Колекція `announcements` (Оголошення)

Документи створяться **автоматично** через додаток (адмін створює в адмінці).  
Просто створи порожню колекцію.

---

#### 📁 Колекція `contacts` (Контакти)

Документи створяться **через адмінку** в додатку.  
Створи порожню колекцію.

Поля, які будуть створені:
- `name` — назва (Сантехнік, Електрик)
- `phone` — номер телефону
- `category` — категорія (Аварійна служба)
- `icon` — емодзі (🔧)
- `order` — порядок сортування

---

#### 📁 Колекція `polls` (Голосування)

Створюються **через додаток** адміном. Порожня колекція.

Поля:
- `question` — питання
- `options` — масив об'єктів `[{text: "Так", votes: []}, {text: "Ні", votes: []}]`
- `active` — `true/false`
- `createdAt` — дата (автоматично)

---

#### 📁 Колекція `neighborPosts` (Сусідські оголошення)

Створюються **через додаток**. Порожня колекція.

Поля:
- `category` — `give`, `find`, `lost`, `other`
- `title` — заголовок
- `content` — текст
- `contact` — контакт
- `author`, `authorApt` — автор
- `createdAt` — дата

---

#### 📁 Колекція `issues` (Проблеми)

Створюються **через додаток**. Порожня колекція.

Поля:
- `title` — назва
- `description` — опис
- `status` — `new`, `in-progress`, `pending`, `resolved`
- `author`, `authorApt` — автор
- `lastComment` — останній коментар (об'єкт)
- `createdAt` — дата

---

#### 📁 Колекція `events` (Події)

Створюються **через додаток** адміном. Порожня колекція.

Поля:
- `title` — назва події
- `description` — опис
- `eventDate` — дата (Timestamp)
- `createdAt` — дата створення

### 6. Залий на GitHub

```bash
cd my-dim-pwa
git init
git add .
git commit -m "Initial commit"
```

Створи репозиторій на [GitHub](https://github.com):

```bash
git remote add origin https://github.com/ТВІЙ_ЛОГІН/my-dim.git
git branch -M main
git push -u origin main
```

### 7. Підключи Vercel

1. Перейди на [https://vercel.com](https://vercel.com)
2. Увійди через GitHub
3. Натисни **Add New** → **Project**
4. Обери репозиторій `my-dim`
5. Налаштування залиш за замовчуванням — Vercel сам визначить, що це статика
6. Натисни **Deploy**

Через хвилину додаток доступний за URL: `https://my-dim.vercel.app`

### 8. Встанови на телефон

1. Відкрий URL додатку в браузері на **телефоні** (Chrome/Safari)
2. Натисни **Share** (Поділитися) → **Add to Home Screen** (На екран додому)
3. Готово! 🎉

## 🆘 Типові проблеми

| Проблема | Рішення |
|----------|---------|
| "Firebase is not defined" | Перевір, чи завантажуються Firebase скрипти в `index.html` |
| "Cannot read property 'apt' of null" | Вийди з додатку (кнопка "Вийти") і зайди знову |
| Не встановлюється на телефон | Відкрий через HTTPS, почекай 30 секунд |
| Помилка "Missing or insufficient permissions" | Скопіюй `firestore.rules` у Firebase Console |
| Не можу увійти | Спочатку створи документ квартири в `apartments` через Firebase Console |
